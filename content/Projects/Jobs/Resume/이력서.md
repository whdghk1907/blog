## 소개

**MSA 아키텍처와 공간정보 기술을 활용한 고성능 GIS API 서버 개발자**  

공간정보 데이터 처리와 API 서버 개발에 전문성을 갖춘 개발자로, **OpenLayers**와 **Geotools**를 활용하여 공간 분석 및 시각화 기능을 설계하고 최적화한 경험이 있습니다.  

이후 최신 공간정보 기술과 클라우드 네이티브 아키텍처에 대한 깊은 관심을 바탕으로 **MORAI**에 합류하여, 자율주행 Unit 개발자들과 협업하며 **HDMap** 및 **SDMap**과 같은 정밀도로지도 포맷을 다루는 프로젝트를 수행했습니다.  

**HDMap 편집 및 Map Editor 개발** 과정에서 사용자 중심의 API 설계와 공간 데이터 처리 최적화를 구현하였으며, 대규모 지도 데이터 연산을 처리하는 **Flask 기반 API 서버 아키텍처 설계** 경험을 보유하고 있습니다.  

또한, **MSA 기반 API 서버 구축**을 통해 **비동기 처리, 병렬 연산, 분산 시스템 설계** 등 대규모 요청을 효과적으로 처리하는 서버 아키텍처를 설계하고 운영한 경험이 있습니다.  

지속적으로 최신 공간정보 기술 트렌드를 탐구하며, 클라우드 및 컨테이너 환경에서 GIS 데이터 처리 성능을 극대화할 수 있는 서버 구조를 설계하고 최적화하는 데 집중하고 있습니다.

### **OpenDRIVE API 모듈 개발 및 최적화  
- **FastAPI 기반 API 서버 전환**: 기존 OpenDRIVE 변환 프로세스를 FastAPI 기반으로 재구성하여 API 호출을 통한 확장성과 재사용성 극대화.  
- **비동기 병렬 처리 아키텍처 구현**: Celery + Redis + Kubernetes 기반의 분산 처리 구조를 설계하여 대용량 변환 요청을 효율적으로 처리.  
- **모듈화 및 코드 관리 최적화**: 기존 mgeo_editor와 변환 로직을 분리하고, Git Submodule 및 독립 레포지토리(pylib_opendrive) 설계로 코드 유지보수성 개선.  
- **API 설계 및 최적화**: 대규모 요청을 처리할 수 있도록 비동기 API를 도입하고, API 응답 속도를 300% 이상 향상.  
- **확장 가능성 고려한 구조 설계**: 단일 사용자용 프로토타입을 개발한 후, 동시 10~20명 이상의 사용자 요청을 처리할 수 있도록 API 서버 스케일링 중.  

### **GeoDjango / PostGIS 기반 GIS 서버 구축
- **PostGIS 데이터베이스 설계**: 지리공간 데이터를 효율적으로 저장 및 관리하기 위해 PostGIS를 도입하고, 공간 인덱싱 및 최적화 적용.  
- **GeoDjango 기반 GIS API 개발**: Django + GeoDjango를 활용하여 지리정보 데이터를 제공하는 API 서버 구축.  
- **GIS 알고리즘 적용**: PostGIS의 공간 연산 기능을 활용해 공간 분석 및 최적화된 경로 탐색 기능 구현.  
- **데이터 모델링 및 성능 최적화**: 기존 데이터 구조와의 호환성을 유지하면서 속성 데이터 간 관계를 매칭하고, View 기반으로 인덱스 최적화 수행.  
- **ISO 20524-2 기반 데이터 관리 모델 설계**: 국제 표준을 반영하여 Belt 개념을 도입한 데이터 관리 및 조회 성능 개선.
